    C++ OOP面向对象开发

    1.1 类 （class）和对象 （objects）

    C++ 类（classes）

    • 创建对象用的蓝图 （blueprint）
    • 用户自定义的数据类型
    • 有成员属性（data）和成员方法（methods）
    • 可以隐藏属性和方法（private）

    • 可以提供公共接口（public）

    • 示例：Account, Student, std:vector, std：：string

    C++ 对象（objects）

    • 表示类的一个具体的实例（Instance）

    • 可以有很多个实例，每个都有独立的身份 • 每个对象都可以使用类中定义的方法

    • Account对象示例

    • Jobs、Alice 的account是Account类的实例

    • 每个实例都有它的余额、可以提现、存钱

    • 声明一个类

    class Account
{
    // 属性
    std::string name;
    double balance;

    // 方法
    bool deposit(double amount); // 存款
    bool withdraw(double amount); // 取款
};
    • 创建类的对象
    Account jobs_account;
Account alice_account;

Account *mary_account = new Account();
delete mary_account;
    • 创建类的对象
    Account jobs_account;
Account alice_account;

// 数组
Account accounts[] {jobs_account, alice_account};

// vector容器
std::vector<Account> accounts_vec {jobs_account};
accounts_vec.push_back(alice_account); // 将alice_account添加到vector容器中

    • 获取类的属性或方法

    • 使用点操作符

      Account jobs_account;

jobs_account.balance; //属性
jobs_account.deposit(1000); //方法

    • 如果是一个指向对象的指针，可以解引用或者使用箭头操作符

      Account *alice_account = new Account();

//要加括号，点的优先级高于解优先级
(*alice_account).balance; // 解引用
(*alice_account).deposit(100.0);

// 用指针访问
alice_account->balance; // 箭头运算符
alice_account->deposit(100.0);

    1.2 public、private、protected访问权限

    public

    • 可以被任何实体访问

    private

    • 只能被本类的方法访问

    protected

    • 可以被本类和子类（继承）的方法访问

    示例

    • private和public冒号后的会自动分配权限
    class Account
{
private:
    // 属性
    std::string name;
    double balance;

public:
    // 方法
    bool deposit(double amount); // 存款
    bool withdraw(double amount); // 取款
};
    • 访问权限调用示例
    Account *alice_account = new Account();

// 以下操作尝试直接访问私有成员变量 'name' 会导致编译错误
// alice_account->name = "Alice"; // 错误，因为 name 是 private 的

// 正确的操作是调用公共成员函数 'deposit'
alice_account->deposit(100.0); // 正确，因为 deposit 是 public 的

delete alice_account; // 释放动态分配的内存

    1.3 实现成员方法

    示例

    • 在类内部实现成员方法
    class Account
{
private:
    // 属性
    double balance {0.0};

public:
    // 方法
    bool setBalance(double amount){
        balance = amount;
        return true;
    };

    double getBalance(){
        return balance;
    };
};
    • 类的外部实现成员方法
    class Account
{
private:
    // 属性
    double balance {0.0};

public:
    // 方法
    bool setBalance(double amount);
    double getBalance();
};

// 类Account的成员函数实现，使用作用域解析运算符 ::
bool Account::setBalance(double amount){
    balance = amount;
    return true;
}

double Account::getBalance(){
    return balance;
}

    1.4 构造函数 （constructor）

    特殊的成员方法

    对象创建的时候自动调用

    适用于实例参数初始化

    函数名和类的名称一致

    无需设置返回类型

    可以被重载（overload）

    示例

    • 构造函数和析构函数示例
    class Account {
private:
    // 属性
    std::string name{"account"};
    double balance{0.0};

public:
    // 构造函数
    Account();
    Account(std::string name);
    Account(double balance);
    Account(std::string name, double balance);
    // 析构函数
    ~Account();
};

    • 构造函数和析构函数被调用

    • Account jobs_account,Account gates_account,Account steve_account,Account mark_account 全部都是在栈上申请的对象

    • 按照出栈-后入先出的顺序释放掉 Account对象 
    {
    Account jobs_account;
    Account gates_account {"Bill Gates"};
    Account steve_account {1000000.0};
    Account mark_account {"Mark Zuckerberg", 100000000.0};
}// 这里调用 4 次析构函数
Account *alice_account = new Account("Alice's account", 1000.0);
delete alice_account; // 这里调用 1 次析构函数

    1.5 析构函数（destructor）

    特殊的成员方法

    函数名和类的名称一致，前面跟着一个~波浪符号

    对象销毁的时候自动调用

    没有参数，没有返回类型

    只有一个析构函数（不能重载）

    适用于释放内存等资源

    1.6 构造函数初始化列表

    示例

    • 构造函数初始化列表
    // 传统的成员变量初始化方式
Account::Account(string name, double balance){
    this->name = name;
    this->balance = balance;
}

// 传统赋值，初值列表初始化成员变量
Account::Account(string name, double balance)
    : name(name), balance(balance) {
}

    1.7 代理构造函数

    • 重载的构造函数很相似
    • 冗余的代码可能会导致错误
    • C++ 允许使用代理构造函数
    • 在一个构造函数初始化列表中调用另一个构造函数

    1.8 拷贝构造函数

    • 当对象被拷贝时，C++必须从已存在的对象复制出一个新的对象
    • 何时使用拷贝构造函数？
    • 以值传递方式传递对象给函数（作参数）
    • 函数以值方式返回对象
    • 使用已存在的对象复制新的对象
    • 如果不提供自己写的copy constructor，编译器会自动生成一个（可能不符合要求）

    示例

    • 值传递给函数
    Account alice_account {"Alice's account", 1000.0};

void printInfo(Account acc) {
    // acc是alice_account的副本
    // 使用acc
    // acc的生命周期结束，析构函数被调用
}

printInfo(alice_account);
    • 以值的方式进行返回
    Account new_account;

Account create_super_account() {
    Account an_account {"An account", 1000.0};
    return an_account; // 返回an_account的副本
}

new_account = create_super_account();
    • 使用已存在的对象复制新的对象
    Account jobs_account {"Jobs", 1000.0};
Account bill_account {jobs_account}; // jobs_account 是一个对象，会调用拷贝构造函数
    • 拷贝构造函数的声明
    Type::Type(const Type &source);

Student::Student(const Student &source);
Account::Account(const Account &source);

    1.9 浅拷贝和深拷贝

    • 如果不提供自己写的copy constructor，编译器会生成默认的

    • 将目标对象的值逐个拷贝过来；

    • 如果是指针，拷贝的是值（指向的地址），而不是指向的对象（浅拷贝） 

      // 编译器生成的拷贝构造函数
Account::Account(const Account &source) {
    this->name = source.name;
    balance = source.balance; // 浅拷贝  此时拷贝的是相同的地址
    this->balance = new double {*source.balance}; //深拷贝 此时拷贝的是对象值
}

    • 在析构函数中释放内存时，其他对象中的指针可能还在指向被释放的资源

    1.10 在类中使用const

    • 常函数

    • 函数名称后加const

    • 函数体内不可以修改类成员属性

      string get_name() const { return name; } //常函数

    • 常对象

    • 声明对象时前面加const

    • 不可以修改常对象的成员属性

    • 仅能调用常函数

       Account const bob_account{"Bob", 200.0}; // 常对象

    1.11 在类中使用static

    • 静态成员变量
    • 所有对象共有同一份数据（数据共享）
    • 在类中声明，类外初始化
    • 静态成员函数
    • 所有对象共享同一个函数
    • 只能访问静态成员变量

    示例

    • 在类中使用 static
    // 类中的静态成员
class Account
{
private:
    static int num_accounts; // 静态成员变量定义

public:
    static int get_num_accounts(); // 静态成员函数声明
    // 更多代码
};

int Account::get_num_accounts()
{   
    // 静态成员方法只能访问静态成员变量
    return num_accounts;
}

    1.12 struct 和 class区别

    示例

    class Account
{
    string name {"none"}; // 默认private
};

struct Student
{
    string name {"none"}; // 默认public
};

Account aa;
aa.name = "bob"; // 错误，无法访问私有成员

Student ss;
ss.name = "alice"; // 可以访问公有成员
