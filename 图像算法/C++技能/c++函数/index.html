      • 1.2 函数原型 (prototype)

      编译链接过程：使用函数前必须先“见过”这个函数

      • 先定义函数原型
        • 避免了小型程序中，不适用于大型项目
      • 使用函数原型
        • 可以编译器暗示函数的存在
        • 声明为前向声明 (forward declaration)
        • 放在程序开头的部分
        • 或者放在头文件(header files) (.h) 中
      • 1.3 参数
      • 当我们传递数据给函数时，其实用的是值传递（pass by value）
      • 数据的拷贝会使得原数据
      • 函数内部的任何不会改变我们传递的原变量
      • 形参（formal parameters）与实参（actual parameters）
        • 形参（formal parameters）：函数定义中声明的参数
        • 实参（actual parameters）：调用函数时传递的参数（arguments）

    • 1.5 函数传参 (按值、引用)

    • 传递数组给函数的声明

    void print_array(int my_numbers[]); // 传递整型数组给函数
    • 数组元素并没有复制！
    • 数组变量名存储了一个指向数组首元素的指针——这个地址才会被复制
    • 函数内部无法知道数组大小（数组大小需要额外传递）

    传递数组

    void print_array(int numbers[]);

int main()
{
    int my_numbers[] = {1, 2, 3, 4, 5};
    print_array(my_numbers);
    return 0;
}

void print_array(int numbers[]){
    // 传入的是数组的首地址
    // 数组的长度是未知的，无法通过数组本身确定
}

    在传入数组的同时，传入数组的大小

    void print_array(int numbers[], size_t size);

int main()
{
    int my_numbers[] = {1, 2, 3, 4, 5};
    print_array(my_numbers, 5);
    return 0;
} 

void print_array(int numbers[], size_t size){
    for (size_t i {0}; i < size; i++){
        cout << numbers[i] << " ";
    }
}

    // 通过将数组声明为常量，我们可以保证数组在函数内部不会被修改（只读）
void print_array(const int numbers[], size_t size){
    for (size_t i = 0; i < size; i++){
        cout << numbers[i] << " ";
    }
    // numbers[0] = 0; // 这一行会报错，因为 numbers 被声明为 const
    cout << endl;
}
    • 有时希望在函数内部改变实参的值
    • 需要实参的地址
    • 类似数组传递方式改变实参，那样实际上是否改变了实参？
    • 可以使用引用传参方式，传递实参的一个别名 (alias)

    引用传递

    #include <iostream>
using namespace std;

// 通过引用，两倍增长
void double_num(int &num)
{
    num *= 2;
}

int main()
{
    // 初始化为10，两倍数字
    int my_num {10};
    cout << "my_num is: " << my_num << endl; // 输出 10
    double_num(my_num);
    cout << "my_num is: " << my_num << endl; // 输出 20

    return 0;
}

    vertor 使用引用传参，节约内存空间

    // 通过引用传递参数
void print_arr(const vector<int> &v) // 接收vector的引用,加一个 const 防止被修改
{
    for (auto i : v){
        cout << i << " ";
    }
    cout << endl;
}

int main()
{
    vector<int> my_vec {10, 20, 30, 40, 50};
    print_arr(my_vec); // 打印 vector 中的元素

    return 0;
}

    • 1.6 函数调用和制

      • 函数调用栈 (function call stack)
        • 栈的特点
          • LIFO (后入先出, last in first out)
          • 入栈 (push) ：在栈顶添加记录
          • 出栈 (pop) ：从栈顶移除记录
        • 栈帧 (stack frame) 或活动记录 (activation record)
          • 每次函数调用时，入栈一个新的活动记录；
          • 函数结束，活动记录从栈顶移除，返回至调用函数的位置；
          • 局部变量 (local variables) 和函数会在栈上分配空间
      • 栈的大小有限，超出会发生栈溢出 (stack overflow)
      内存区域 类型 描述
      堆 动态（Free memory） 用于动态分配内存，如使用 new 或 malloc
      栈 本地变量（Local variables） 函数内部声明的变量和函数的调用栈
      全局区 全局变量（Global Variable）, 静态变量（static variable） 程序的全局变量和静态变量
      代码区 一次性代码 存储程序的机器代码或执行指令

      这张图片显示了一个递归函数的示例和它在调用时的函数调用栈的可视化。

      代码部分是一个计算阶乘的C++函数：

      #include <iostream>
using namespace std;

unsigned long long factorial(unsigned long long n) {
    if (n == 0)
        return 1; // 基本案例，0的阶乘是1
    return n * factorial(n - 1); // 递归调用
}

int main() {
    cout << factorial(3) << endl; // 6
    // cout << factorial(8) << endl; // 40320
    // cout << factorial(12) << endl; // 479001600
    // cout << factorial(20) << endl; // 2432902008176640000
    return 0;
}

      图片中的绿色框表示函数调用栈，每个框代表一次factorial函数的调用。例如，factorial(3)会在栈上产生三个框，每个框对应一个调用factorial(n)，其中n依次为3、2、1。

      以下是函数调用栈的表格表示：

      调用 返回值 栈的状态
      factorial(3) 3 * factorial(2) n = 3
      factorial(2) 2 * factorial(1) n = 2
      factorial(1) 1 * factorial(0) n = 1
      factorial(0) 1 n = 0

      当到达基本案例n == 0时，函数返回1，然后逐层返回并计算最终结果。
